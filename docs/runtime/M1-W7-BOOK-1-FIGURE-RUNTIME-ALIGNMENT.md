# M1-W7 Book 1 Figure-to-Runtime Alignment

This closure aligns Figures 1A–5E with the existing Runtime Journey without adding a Runtime stage. Figures 1A, 1B, 2A, and 5A–5E are conceptual references mapped to existing contracts; they are not additional executable stages.

## Parts 1 and 2

Figure 1A's R1–R9 Runtime Regions and Figure 3A's R1–R9 Runtime Capabilities remain separate representations. A Region describes where Reality is structurally organized or operating; a Capability describes which response capacity is currently available. The frozen fields remain `runtimeRegions` and `runtimeCapabilities` respectively.

Figure 1B maps Structure, Connectivity, Network, Direction, Position, and Navigation Potential onto Reconstruction, Reading, and Navigation. It does not introduce a Navigation Physics stage.

Figure 2A maps the Projection Pipeline onto the Evidence Boundary and Provider Router. Provider output may assist interpretation, but cannot rewrite or upgrade the user's observed evidence.

Figure 3A is represented as Entry → G1–G16 → Active Question → R1–R9 Capability → optional D1–D12 Driver Priority → bounded Navigation. Drivers require explicit Reader evidence and are never inferred automatically.

Figure 3B separates Runtime Coordinate from biological or carrier organization. Its canonical coordinates are Temporal, Condition, Directional, Carrier, and Relational. Directional Coordinate is not the removed Directional Signature.

Figures 4A and 4B establish separate Carrier Organization and Carrier Configuration representations. Carrier Runtime Style is a dynamic configuration output, not a fixed Signature Family. The six removed Signature Families are not regenerated.

Figures 3C, 4C, and 4D remain represented by the frozen Review, Memory, Continuity, Recovery, Revision, and Lineage contracts. Their infrastructure persistence is implemented in M2; this alignment does not reopen the Runtime stage model.

## Part 5

Figures 5A–5E map Experience to Reading, Agency to Navigation and Review, and Identity Continuity to Memory and Continuity. Their C1–C5 labels describe conceptual engines and outputs, not new Runtime Journey stages.

Compression in Figures 1A, 5B, and 5D is a mechanism within formation, expression, or identity organization. It is not a replacement label for G9; the canonical grammar remains G9 Expression.

Agency is the canonical term and `agency_style` is the canonical contract field. "Action Style" is retained only as customer-facing compatibility wording where the figure uses it.

Figure 5D's "Identity Runtime Signature" is an identity output descriptor. It is not any of the removed Structural, Relational, Navigational, Resource, Directional, or Temporal Carrier Signature families.

The deprecated Entry target `carrier_signatures` is accepted only when restoring an old session and is normalized to `runtime_conditions`. New sessions use `runtime_conditions` exclusively.
