# M1-W7 Book 1 Figure-to-Runtime Alignment

This closure aligns Figures 3A–4D with the existing Runtime Journey without adding a Runtime stage.

Figure 3A is represented as Entry → G1–G16 → Active Question → R1–R9 Capability → optional D1–D12 Driver Priority → bounded Navigation. Drivers require explicit Reader evidence and are never inferred automatically.

Figure 3B separates Runtime Coordinate from biological or carrier organization. Its canonical coordinates are Temporal, Condition, Directional, Carrier, and Relational. Directional Coordinate is not the removed Directional Signature.

Figures 4A and 4B establish separate Carrier Organization and Carrier Configuration representations. Carrier Runtime Style is a dynamic configuration output, not a fixed Signature Family. The six removed Signature Families are not regenerated.

Figures 3C, 4C, and 4D remain represented by the frozen Review, Memory, Continuity, Recovery, Revision, and Lineage contracts. Their infrastructure persistence is implemented in M2; this alignment does not reopen the Runtime stage model.

The deprecated Entry target `carrier_signatures` is accepted only when restoring an old session and is normalized to `runtime_conditions`. New sessions use `runtime_conditions` exclusively.
